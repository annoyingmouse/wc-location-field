<wc-location-field>

A self-contained web component for picking a location. Provides address search via Nominatim (OpenStreetMap), an optional Leaflet map, browser geolocation, and optional What3Words integration. No framework dependencies.

Installation

<script src="https://unpkg.com/@annoyingmouse/wc-location-field/wc-location-field.js" defer></script>

That's it — styles and Leaflet (when show-map is used) are loaded automatically.

Or via CDN as a module:

<script type="module" src="https://cdn.skypack.dev/@annoyingmouse/wc-location-field/wc-location-field.js"></script>

Attributes

  • label — field label text (reactive)
  • placeholder — input placeholder text (reactive)
  • center-lat / center-lng — map centre and Nominatim search bias
  • show-map — renders an interactive Leaflet map below the search box
  • w3w-key — What3Words API key; enables ///word.word.word search

Public API

  • lat, lng, address — read-only state getters
  • value — W3W address if resolved, else formatted address, else raw input
  • prefill({ address, lat, lng }) — programmatically set location without firing an event
  • clear() — reset all state

The component fires a location-change CustomEvent (bubbles) on every resolved location, with detail: { lat, lng, address, w3w }.

Examples

Search only

Minimal usage — just a labelled search box, no map.

<wc-location-field
  label="From"
  placeholder="e.g. Paddington Station"
></wc-location-field>

With map

Adds an interactive Leaflet map — tap or drag the marker to set a precise location.

<wc-location-field
  label="Meeting point"
  show-map
  center-lat="51.505"
  center-lng="-0.09"
></wc-location-field>

Search bias

Setting center-lat and center-lng biases Nominatim results toward that area.

<wc-location-field
  label="Destination"
  show-map
  center-lat="52.2053"
  center-lng="0.1218"
></wc-location-field>

Public API

Use prefill(), clear(), and the value / lat / lng getters from JavaScript.

const field = document.querySelector('#demo-api');

// Prefill
field.prefill({ address: 'Buckingham Palace', lat: 51.5014, lng: -0.1419 });

// Clear
field.clear();

// Read
console.log(field.value, field.lat, field.lng);

location-change event

The component fires a location-change event on every resolved location. Set a w3w-key attribute to also enable What3Words search (///word.word.word).

Events received:

Pick a location above to see events…
field.addEventListener('location-change', e => {
  const { lat, lng, address, w3w } = e.detail;
});

Theming

The component uses no Shadow DOM, so CSS custom properties cascade in from any ancestor. Override them on the element or a wrapper: 

wc-location-field {
  --lf-map-height: 300px;
  --lf-w3w-color: #e11f26;
}

Available custom properties:

  • --lf-map-height — height of the Leaflet map (default: 240px)
  • --lf-w3w-color — colour of the /// What3Words prefix (default: #e11f26)

All other visual styling — input borders, button appearance, fonts, colours — is intentionally left unstyled so it inherits from your own stylesheet or design system.

Accessibility

The search input uses the ARIA combobox pattern (role="combobox", aria-expanded, aria-haspopup="listbox", aria-controls). Coordinates are announced via an aria-live="polite" region. The GPS button has an accessible label.
